# 节点配置UI对比

## 升级前后对比

### 升级前
所有节点配置都是一个简单的JSON文本框：
```
┌─────────────────────────────────────┐
│ 配置 JSON                           │
├─────────────────────────────────────┤
│ {                                   │
│   "api_key": "",                    │
│   "model": "gpt-4o-mini",          │
│   "temperature": 0.1,               │
│   "chunk_size": 2000,               │
│   ...                               │
│ }                                   │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

**问题：**
- ❌ 需要手写JSON，容易出错
- ❌ 不知道有哪些配置项
- ❌ 不知道每个配置项的含义
- ❌ 不知道合理的取值范围
- ❌ 没有验证，错误只在执行时才发现

## 各节点配置对比

### 1. LLMJson V2 节点

#### 配置分组
- **API配置** (2个字段)
  - API密钥 - 密码输入框
  - API基础URL - 文本输入框

- **模板配置** (1个字段)
  - 模板选择 - 下拉框（通用/洪涝灾害）

- **模型配置** (3个字段)
  - 模型名称 - 下拉框（4个选项）
  - 生成温度 - 数字输入器（0-1，步长0.1）
  - 最大输出长度 - 数字输入器（100-128000）

- **处理配置** (2个字段)
  - 文档分块大小 - 数字输入器（500-10000）
  - 处理表格内容 - 开关按钮

- **高级配置** (3个字段)
  - 请求超时时间 - 数字输入器（10-300秒）
  - 最大重试次数 - 数字输入器（0-10次）
  - 文件编码 - 下拉框（UTF-8/GBK/GB2312）

### 2. LLMJson 节点

#### 配置分组
- **API配置** (2个字段)
  - API密钥 - 密码输入框
  - API基础URL - 文本输入框

- **模型配置** (3个字段)
  - 模型名称 - 下拉框（4个选项）
  - 生成温度 - 数字输入器（0-1）
  - 最大token数 - 数字输入器（100-128000）

- **文档处理** (5个字段)
  - 文档分块大小 - 数字输入器（500-10000）
  - 分块重叠 - 数字输入器（0-1000）
  - 并行工作线程数 - 数字输入器（1-16）
  - 启用并行处理 - 开关按钮
  - 处理表格内容 - 开关按钮

- **输出设置** (2个字段)
  - 输出目录路径 - 文本输入框
  - 保存中间结果 - 开关按钮

- **高级配置** (3个字段)
  - 请求超时时间 - 数字输入器（10-300秒）
  - 最大重试次数 - 数字输入器（0-10次）
  - 文件编码 - 下拉框（4个选项）

### 3. Json2Graph 节点

#### 配置分组
- **数据库配置** (3个字段)
  - Neo4j URI - 文本输入框
  - Neo4j 用户名 - 文本输入框
  - Neo4j 密码 - 密码输入框

- **处理配置** (2个字段)
  - 存储模式 - 下拉框（SKG基础/STKG增强）
  - 处理器配置列表 - JSON编辑器（带格式化）

- **高级选项** (1个字段)
  - 导入前清空数据库 - 开关按钮（危险操作提示）

### 4. VectorIndexer 节点

#### 配置分组
- **基础配置** (1个字段)
  - 向量集合名称 - 文本输入框（1-100字符）

- **处理配置** (3个字段)
  - 文本分块大小 - 数字输入器（100-2000）
  - 分块重叠大小 - 数字输入器（0-500）
  - 使用jieba优化 - 开关按钮

- **元数据配置** (3个字段)
  - 文档类型 - 下拉框（6个选项）
  - 文档来源 - 文本输入框
  - 文档语言 - 下拉框（5个选项）

- **高级选项** (1个字段)
  - 覆盖已存在文档 - 开关按钮（危险操作提示）

## UI控件类型统计

### 使用的控件类型
1. **文本输入框** - 普通字符串字段
2. **密码输入框** - API密钥、数据库密码等敏感信息
3. **数字输入器** - 数值参数，支持范围和步长
4. **下拉选择框** - 有限选项的字段
5. **开关按钮** - 布尔值字段
6. **JSON编辑器** - 复杂对象/数组字段

### 控件选择规则
- 字段名包含 `key`/`password`/`secret` → 密码输入框
- 字段类型为 `bool` → 开关按钮
- 字段类型为 `int`/`float` → 数字输入器
- 字段有 `options` → 下拉选择框
- 字段类型为 `list`/`dict` → JSON编辑器
- 其他 → 文本输入框

## 验证规则

### 自动验证
- **必填字段** - 标记为必填，提交时验证
- **数值范围** - 自动限制最小值和最大值
- **字符长度** - 限制最小和最大长度
- **正则表达式** - 自定义格式验证

### 实时反馈
- 输入时实时验证
- 错误信息即时显示
- 范围外的值自动调整
- 不合法的输入被阻止

## 开发体验提升

### 配置简单
只需在Pydantic Field中添加 `json_schema_extra`：
```python
api_key: str = Field(
    default="",
    description="API密钥",
    json_schema_extra={
        'group': 'api',
        'format': 'password'
    }
)
```

### 自动推断
即使不配置UI元数据，系统也会自动推断：
- 字段名包含 `password` → 自动使用密码框
- 字段名包含 `chunk` → 自动归入处理配置组
- 字段类型为 `bool` → 自动使用开关

### 类型安全
- 利用Pydantic的类型验证
- 前端表单自动匹配后端类型
- 减少类型转换错误
