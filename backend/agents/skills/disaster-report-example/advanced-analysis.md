# 高级分析方法

本文档提供更深入的灾害分析方法和技巧。

## 多维度对比分析

### 1. 跨年度对比
使用 `compare_entities` 工具对比不同年份的同类灾害：
```
比较 2020年、2021年、2022年 三年的洪涝灾害受灾情况
```

关注指标：
- 受灾人口变化趋势
- 经济损失对比
- 应急响应效率提升

### 2. 跨地区对比
对比不同地区在同一灾害中的表现：
- 灾害强度差异（降雨量、水位）
- 损失程度差异
- 防灾能力评估

### 3. 时空关联分析
结合 `analyze_temporal_pattern` 和 `get_spatial_neighbors`：
- 分析灾害传播路径
- 识别高风险区域
- 评估空间防护措施有效性

## 因果链深度挖掘

### 1. 多跳因果追踪
使用 `find_causal_chain` 的 `max_depth` 参数：
```python
# 追踪 3 层因果关系
find_causal_chain(
    start_event="强降雨",
    max_depth=3,
    direction="forward"
)
```

### 2. 双向因果分析
结合正向和反向追踪：
- `direction="forward"` → 追踪影响
- `direction="backward"` → 追溯原因
- `direction="both"` → 完整因果网络

### 3. 关键节点识别
在因果链中识别：
- **触发点**：导致级联反应的初始事件
- **放大器**：加剧灾害的中间因素
- **缓冲点**：可以干预阻断的环节

## 数据整合策略

### 1. 多源数据融合
整合来自不同工具的数据：
1. 图谱查询 → 基础事实数据
2. 预案检索 → 规范性指导
3. 时序分析 → 趋势规律
4. 空间分析 → 地理分布

### 2. 数据交叉验证
使用多个查询验证关键数据：
```
# 方法 1：直接查询受灾人口
query_knowledge_graph_with_nl("2023年广西洪涝受灾人口")

# 方法 2：通过聚合统计验证
aggregate_statistics(
    entity_type="地点",
    attribute="受灾人口",
    aggregation="sum",
    time_range=["2023-01-01", "2023-12-31"]
)
```

### 3. 缺失数据处理
当数据不完整时：
- 明确标注数据缺口
- 使用相似案例估算（注明）
- 提出数据补充建议

## 可视化最佳实践

### 1. 选择合适的图表类型
- **折线图**：时序趋势（降雨量、水位变化）
- **柱状图**：地区对比（各市受灾情况）
- **饼图**：构成占比（损失类型分布）
- **热力图**：空间密度（受灾强度分布）

### 2. 图表设计原则
- 标题清晰：说明图表展示的内容
- 轴标注完整：包含单位和范围
- 颜色有意义：红色表示警告，绿色表示安全
- 数据标签：关键数据点直接标注数值

### 3. 多图表组合
复杂分析使用组合图表：
- 主图 + 子图：整体趋势 + 局部放大
- 对比图：并排展示不同维度
- 时间轴 + 地图：展示时空演化

## 报告撰写技巧

### 1. 结构化叙述
- 先结论后细节：执行摘要置顶
- 分层展开：从宏观到微观
- 逻辑清晰：因果关系明确

### 2. 数据驱动
- 每个结论都有数据支撑
- 引用具体数值和来源
- 避免模糊描述（"大量"→"XX万"）

### 3. 可操作性
- 建议具体可执行
- 引用相关预案条款
- 区分短期/中期/长期措施

## 常见陷阱

### 1. 数据陷阱
- ❌ 混淆"受灾人口"和"转移人口"
- ❌ 累计数据与单次数据混用
- ❌ 忽略数据的时间范围

### 2. 分析陷阱
- ❌ 相关性不等于因果性
- ❌ 孤立看待数据，忽视背景
- ❌ 过度解读不确定数据

### 3. 表达陷阱
- ❌ 使用绝对化表述（"完全"、"必然"）
- ❌ 编造未查询到的数据
- ❌ 忽略数据来源的可靠性
